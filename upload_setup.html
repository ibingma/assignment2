Uploading and Testing

Steps to Upload the Website on a Server

  1. Select a Web Hosting Provider
    • This is basically renting a storage space in a server that will store the website and make it accessible anytime through the Web.
    • There are several types of hosting services1 – dedicated, cloud or shared.
  2. Obtain a Domain Name
    • Domain name is the name we would like to have for our website. It has to registered and unique meaning no duplicates of someone elses website’s name.
    • It can be purchase from a domain registrar (e.g., GoDaddy, Namecheap).
    • GitHub Pages provides a free service to upload and publish a website which uses a given subdomain (e.g.,https://ibingma.github.io/assignment2/index.html)
  3. Upload Files to the Server
    • All files needed by the website must be uploaded to the server. Two common ways to do this are through:
      • FTP (File Transfer Protocol):
        • Use an FTP client (e.g., FileZilla, Cyberduck).
      • Control Panel (cPanel/File Manager):
        • This is available on the web hosting provider’s control panel (usually cPanel).
  4. Publish the website
    • Once the files are uploaded to the server, follow the hosting provider’s specific settings and instructions to publish the website. The exact steps may vary depending on the hosting provider.
    • Make sure your main page (e.g., index.html or index.php) is in the root directory so that it automatically loads when visitors access your website's URL.

Steps to Test Web Pages

  1. Visit the Domain
    • Open a web browser and go to your website’s URL (e.g., https://www.yourdomain.com or https://username.github.io).
    • Ensure that the page loads without errors.
  2. Check All Links
    • Click on internal links (navigation menus, buttons, etc.) to ensure they direct users to the correct pages.
    • Test external links to verify they open in new tabs if required.
  3. Test Images, Styles, and Scripts
    • Make sure all images are displayed correctly and that any styles (CSS) and scripts (JavaScript) are functioning as expected.
  4. Test on Multiple Devices and Browsers
    • View your site on different devices (desktop, mobile, tablet) and browsers (Chrome, Firefox, Safari, Edge) to ensure that it looks and works well across all platforms.
    • Use browser developer tools to check for any console errors or loading issues.
  5. Check Responsiveness
    • Ensure that the site is responsive (adjusts well to different screen sizes) if implementing a responsive design.
